Cabeçalhos & corpo

Escala tipográfica

Todo o grid de tipografia é baseado em duas variáveis Less no arquivo variables.less: @baseFontSize e @baseLineHeight. O primeiro é o font-size base usado proporcionalmente e o segundo é o line-height base

Nós usamos estas variáveis, e mais alguma matemática para criar as margens, paddings e line-height de todos nossos tipos e mais.

Inline

Wrap inline snippets of code with <code>.

Basic block

Use <pre> para múltiplas linhas de código, certifique-se de escapar qualquer caractere especial para renderizar corretamente

<p>Texto de exemplo aqui</p>

<pre>
  &lt;p&gt;Texto de exemplo aqui&lt;/p&gt;
</pre>

Nota: Certifique-se de mantes o código dentro da tag <pre> a extrema esquerda, por que o browser irá renderizar tabulação.

Você pode opcionalmente adicionar a classe .pre-scrollable que irá escolher um max-height of 350px e provê uma barra de progresso no eixo y

Take the same <pre> element and add two optional classes for enhanced rendering.

</pre>

Formulários

HTML e CSS flexível

A melhor parte sobre formulários é que o Bootstrap é seus inputs e controles ficam ótimos não importa como você faça a marcação HTML. Nenhuma marcação supérfula é necessária, mas provemos padrões para estes que são requeridos

Layouts mais complexos tem classes sucintas e escaláveis para estilizar e você fica a par de todos os passos

Quatro layouts de exemplo incluídos

O Bootstrap vem com suporte para quatro tipos de layout

  • Vertical (padrão)
  • Busca
  • Inline
  • Horizontal

Diferentes tipos de layouts de formulários são necessários para mudança na marcação HTML, mas os controles por si só permanecem o mesmo

Controle de estados e mais

Os formulários do Bootstrap incluem estilos para todos os controles bases como input, textarea, e selects. Mas também vem com um número de componentes customizados como inputs apendados e prependados e suporte para listas de checkboxes

Estados como erro, aviso, e sucesso estão incluídos para cada tipo de controle de formulário. Também está incluído todos estilos para controles desabilitados

Quatro tipos de formulários

O Bootstrap vem com uma marcação HTML simples e estilos para quatro tipos de comuns de formulários para web

Nome Classe Descrição
Vertical (padrão) .form-vertical (Não requerido) Estagnado, alinhado a esquerda de rótulos e controles
Inline .form-inline Alinhado a esquerda e controles inline-block para estilos compactos
Busca .form-search Uma borda arredondada extra para dar o aspecto de busca
Horizontal .form-horizontal Flutuando a esquerda, títulos alinhados a direita na mesma linha dos controles

Formulários de exemplo Usando só controles de formulários, sem nenhuma marcação html extra

Formulário de busca

Refletindo o padrão do Webkit, adicione .form-search para formulários de busca arredondados

<form class="well form-search">
  <input type="text" class="input-medium search-query">
  <button type="submit" class="btn">Busca</button>
</form>

Formulário inline

Inputs são elementos bloco para começar. Para .form-inline e .form-horizontal, nós usamos inline-block

<form class="well form-inline">
  <input type="text" class="input-small" placeholder="Email">
  <input type="password" class="input-small" placeholder="Senha">
  <label class="checkbox">
    <input type="checkbox"> Lembre-se de mim
  </label>
  <button type="submit" class="btn">Entrar</button>
</form>

Botões para ações

Como uma convenção, botões sempre devem ser usados para ações enquanto links devem ser usado para objetos. Por exemplo, "Download" deveria ser um botão enquanto "atividades recentes deveria ser um link.

Estilos de botões podem ser aplicaods para qualquer coisa dentro da classe .btn. Entretanto, tipicamente você quer aplicar isto em elementos <a> e <button>.

Compatibilidade Cross Browser

O IE9 não corta gradiente de fundo em bordas arredondadas, então nós removemos. Outra coisa, IE9 desabilita elementos button, renderizando um texto cinza com uma sombra no texto que não podemos corrigir.

Múltiplos tamanhos

Botões elegantes grandes e pequenos? Adicione .btn-large, .btn-small, ou .btn-mini para dois tamanhos adicionais


Estado desabilitado

Para botões desabilitados, adicione a classe .disabled para links e o atributo disabled para elementos <button>

Link primário Link

Atenção! Nós usamos .disabled como uma classe utilitária, similar a classe .active, nenhum prefixo é necessário

Construído como um sprite

ao invés de fazer um request extra, nós compilamos em um sprite—com várias images em um arquivo que usa o CSS para posicioná-los através do background-position. Este é o mesmo método usado no twitter.com e funciona muito bem para nós.

Todas classes de ícones tem um prefixo .icon- para namespace e escopo. Isto irá evitar conflitos com outros componentes.

Glyphicons nos forneceu o uso da sua ferramenta e colocamos um link deles na nossa documentação. Por favor considere fazer o mesmo no seu projeto.

Como usar

O Bootstrap utiliza uma tag <i> para todos ícones. Para usar, coloque o seguinte código onde desejar:

<i class="icon-search"></i>

Há também estilos disponíveis para ícones (brancos) invertidos, prontos para usar somente com uma classe extra

<i class="icon-search icon-white"></i>

Há 120 classes para escolha dos ícones. Simplesmente adicione uma tag <i> com a classe correta e pronto. Você pode encontrar a lista completa no arquivo sprites.less ou aqui neste documento.

Atenção! Quando usar além de strings de texto, como um botão ou links de navegação, tenha certeza de deixar um espaço depois da tag para espaçamento apropriado

Casos de uso

Ícones são ótimos, mas onde usá-los? Aqui estão algumas idéias:

  • Como apelo visual para sua navegação na barra lateral
  • Para uma navegação guiada puramente por ícones
  • Para botões para ajudar a lembrar o significado de uma ação
  • Com links para compartilhar um contexto do destino do usuário

Essencialmente, qualquer lugar você pode colocar uma tag <i>, você pode colocar um ícone

Exemplos

Use-os como botões, grupos de botões para uma barra de ferramentas, navegação, ou inputs de formulários

Estilos globais para reset CSS, tipografia e links

Requer o doctype de HTML5

O Bootstrap faz o uso de elmentos HTML e propriedades CSS que requer o uso de um doctype HTML5. Tenha certeza de inclir isto em cada página Bootstratiada em seu projeto

<!DOCTYPE html>
<html lang="en">
  ...
</html>

Tipografia e links

Com o arquivo scaffolding.less, nós escolhemos uma exibição global para tipografia e links. Especificadamente, nós:

  • Removemos as margens do body
  • Escolhemos o background-color: white; no body
  • Usamos o @baseFontFamily, @baseFontSize, e @baseLineHeight atributos para a tipografia básica
  • Escolhemos cores globais de link via @linkColor e aplicamos linha abaixo do link com :hover

Reset via Normalize

Com o Bootstrap 2, o reset tradicional de elementos foi feito via Normalize.css, um projeto do Nicolas Gallagher que também usou o poder do HTML5 Boilerplate.

O novo reset pode ainda ser encontrado no reset.less, mas com muitos elementos removidos para brevidade e eficácia

Aninhando colunas

Com o sistema de grid estático(não fluido) no Bootstrap, aninhamento é fácil. Para aninhar seu conteúdo, simplesmente adicione um novo .row e as colunas em .span* colunas de uma classe existente .span*.

Exemplo

Linhas aninhadas devem incluir uma coleção de colunas que adiciona o número do seu pai. Por exemplo, duas colunas aninhadas .span3 devem ser colocadas dentro de uma .span6.

Nível 1 da coluna

      <div class="span6">Nível 2</div>

Porcento, não pixels

O sistema de grid fluid usa porcento para largura das colunas ao invés de pixels fixo. Ele também a mesma variação responsive como o sistema de grid fixo, certificando de que as proporções permanecerão em resoluções diferentes e dispositivos

Linhas fluidas

Qualquer linha fluida pode ser modificada mudando .row para .row-fluid. A coluna fica da mesma maneira, sendo super direto trocar entre layout fixos e fluidos

Marcação

<div class="row-fluid">
  <div class="span4">...</div>
  <div class="span8">...</div>
</div>

Teste de fluido

Aninhando com grids fluids é um pouco diferente: o número de colunas aninhadas não precisa ser igual ao pai. Ao invés disto, suas colunas são reiniciadas em cada nível por cada linha tem 100% da sua coluna pai.

Customização do grid

Variável Valor padrão Descrição
@gridColumns 12 Número de colunas
@gridColumnWidth 60px Largura da coluna
@gridGutterWidth 20px Espaço negativo entre colunas
@siteWidth Soma computada de todas colunas de espacamento(gutter) Numéro de colunas e espaçamento para o mixin de .container-fixed()

Variáveis no LESS

Construído dentro do Bootstrap uma útil coleção de variáveis para customizar o grid system de 940px, documentado acima. Todas variáveis para grid estão armazenados em uma variável em variables.less

Como customizar

Modificar o grid significa mudar as três variáveis @grid* e recompilar o código. Mudando as variáveis de grid em variables.less e usá-los em uma das quatro maneiras de recompilar. Se você está adicionando mais colunas, tenha certeza de adicionar o CSS para elas em grid.less

Continue responsivo

Customização do grid só funciona no nível padrão, no grid de 940px. Para manter o aspecto do Bootstrap responsivo, nós também temos uma customização do grid em responsive.less

Layouts templates básicos para criar páginas

Fixed layout

The default and simple 940px-wide, centered layout for just about any website or page provided by a single <div class="container">.

<body>
  <div class="container">
    ...
  </div>
</body>

Layout fluido

<div class="container-fluid"> gives flexible page structure, min- and max-widths, and a left-hand sidebar. It's great for apps and docs.

<div class="container-fluid">
  <div class="row-fluid">
    <div class="span2">
      <!--Sidebar content-->
    </div>
    <div class="span10">
      <!--Body content-->
    </div>
  </div>
</div>

Responsive design Media queries for various devices and resolutions

What they do

Media queries allow for custom CSS based on a number of conditions—ratios, widths, display type, etc—but usually focuses around min-width and max-width.

  • Modify the width of column in our grid
  • Stack elements instead of float wherever necessary
  • Resize headings and text to be more appropriate for devices

Use media queries responsibly and only as a start to your mobile audiences. For larger projects, do consider dedicated code bases and not layers of media queries.

Supported devices

Bootstrap supports a handful of media queries in a single file to help make your projects more appropriate on different devices and screen resolutions. Here's what's included:

Label Layout width Column width Gutter width
Smartphones 480px and below Fluid columns, no fixed widths
Smartphones to tablets 767px and below Fluid columns, no fixed widths
Portrait tablets 768px and above 42px 20px
Padrão 980px and up 60px 20px
Large display 1200px and up 70px 30px

Requires meta tag

To ensure devices display responsive pages properly, include the viewport meta tag.

<meta name="viewport" content="width=device-width, initial-scale=1.0">

Using the media queries

Bootstrap doesn't automatically include these media queries, but understanding and adding them is very easy and requires minimal setup. You have a few options for including the responsive features of Bootstrap:

  1. Use the compiled responsive version, bootstrap-responsive.css
  2. Add @import "responsive.less" and recompile Bootstrap
  3. Modify and recompile responsive.less as a separate file

Why not just include it? Truth be told, not everything needs to be responsive. Instead of encouraging developers to remove this feature, we figure it best to enable it.

  // Landscape phones and down
  @media (max-width: 480px) { ... }

  // Landscape phone to portrait tablet
  @media (max-width: 768px) { ... }

  // Portrait tablet to landscape and desktop
  @media (min-width: 768px) and (max-width: 980px) { ... }

  // Large desktop
  @media (min-width: 1200px) { .. }

Responsive utility classes

What are they

For faster mobile-friendly development, use these basic utility classes for showing and hidding content by device.

Quando usar

Use on a limited basis and avoid creating entirely different versions of the same site. Instead, use them to complement each device's presentation.

For example, you might show a <select> element for nav on mobile layouts, but not on tablets or desktops.

Support classes

Shown here is a table of the classes we support and their effect on a given media query layout (labeled by device). They can be found in responsive.less.

Class Phones 480px and below Tablets 767px and below Desktops 768px and above
.visible-phone Visible Hidden Hidden
.visible-tablet Hidden Visible Hidden
.visible-desktop Hidden Hidden Visible
.hidden-phone Hidden Visible Visible
.hidden-tablet Visible Hidden Visible
.hidden-desktop Visible Visible Hidden
